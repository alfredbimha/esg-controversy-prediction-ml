ESG Controversy Event Prediction

Machine Learning for Corporate ESG Risk Forecasting

Languages: Python Category: Sustainable Finance Analytics
Research Question Can machine learning models predict which companies are most likely to experience ESG controversies based on observable firm characteristics?
Methodology

Binary classification using XGBoost, Random Forest, and Logistic Regression. Features include lagged ESG scores, financial ratios, sector, and governance characteristics. Model performance evaluated via AUC-ROC, precision-recall, and SHAP feature importance.

Data Collection & Sources
Source: Simulated ESG controversy events modelled on RepRisk/MSCI structure
Target: Binary — controversy event within next 12 months (1/0)
Features: 25 predictors including ESG scores, financial ratios, board diversity, audit quality
Sample: 2,000 firm-years with ~15% positive rate (controversies)
Key Methods
XGBoost Classifier
Gradient-boosted trees handling non-linear relationships and feature interactions — strong baseline for tabular prediction tasks
SHAP Feature Importance
SHapley Additive exPlanations decompose each prediction into feature contributions — provides model interpretability
Cross-Validation
Stratified 5-fold CV with time-aware splits — prevents data leakage from future information
Key Findings
1
XGBoost achieves AUC-ROC of ~0.78, outperforming logistic regression (~0.68) and random forest (~0.74)
2
SHAP analysis reveals governance quality and previous controversy history as the two strongest predictors
3
Industry sector is a significant predictor — extractive and consumer-facing firms have 2–3× higher controversy probability
